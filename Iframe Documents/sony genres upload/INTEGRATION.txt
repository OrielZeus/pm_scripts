=============================================

Folder: "sony genres upload"
Collection: 23 (genres)

-------------------

  Placeholder: {API_HOST}

Full URLs — Collection API (called from PHP via callApiUrlGuzzle, not browser)
------------------------------------------------------------------------------
These are relative to {API_HOST} (same base as above).

  List records:   {API_HOST}/collections/23/records?page={n}&per_page={m}
  Export job:     {API_HOST}/collections/23/export          (POST)
  Download file:  {API_HOST}/collections/23/download/{hash}   (from export job / UI; hash from PM)

What each script does
---------------------
1) Sony Genres Upload Main Iframe.php
   Tabbed UI: collection, Excel import, comparison dashboard (New / To update / Collection only / OK), backup/apply.
   Default request_id 90235. Calls Get Collection, Save JSON backup, Upload file, Export, Apply PSTools.
   Comparison matches each Excel row to the collection in order: (1) Local Genre Key === collection value;
   (2) else normalized Local Genre Name vs collection content/name (case- and accent-insensitive).
   Apply sends optional record_id when a row was matched by name so the server updates the correct record.

2) Sony Genres Get Collection Data.php
   POST returns { success, data: [ { record_id, name, value, row_id, content, raw } ] }.
   Embeds callApiUrlGuzzle + encodeSql at file bottom. Needs API_HOST and API_TOKEN.

3) Sony Genres Export Backup.php
   POST triggers export via callApiUrlGuzzle. Optional file attach uses raw Guzzle GET.

4) Sony Genres Apply Collection.php
   replace: delete all collection records, then POST each Excel row. adjust: PATCH only existing keys (no creates). mix: POST missing + PATCH changed.
   excel_rows may include record_id (from the iframe when the row was matched by label); that ID takes precedence over value lookup.

5) Sony Genres Save Backup To Request.php
   PUT /requests/{id} merging data.collection_backup (full collection snapshot).

6) Sony Genres Upload File To Request.php
   Multipart POST /requests/{id}/files so the Excel is stored on the request (same idea as Upload Documents Script).

PM registration
---------------
Create one PSTools script per PHP file. Set each script’s slug so the path matches the “Full URL” lines above, or change the slugs in Sony Genres Upload Main Iframe.php ($pstoolsPaths) to match what you registered.

Browser token
-------------
If PSTools require Authorization from the iframe, set bearerToken in the main iframe JS
(see Sony Genres Upload Main Iframe.php), same pattern as Upload Documents Script.

Export / download
-----------------
Export may be async (HTTP 202). If the JSON has no download URL, use PM Collections UI or
{API_HOST}/collections/23/download/{hash} when you have the hash.

Request data
------------
  collection_backup — JSON snapshot from “Save JSON backup to request” (includes records[] with raw data JSON).

Excel
-----
Recognized headers (normalized): Local Genre Key (or Country Local Genre Key), Local Genre Name,
Is Active (or Is Active Local Genre), optional Global Genre, optional Row Id.
